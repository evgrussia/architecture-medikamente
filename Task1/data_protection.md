# Классификация данных и способы их защиты

Документ описывает, какие данные подлежат защите в «Медикаменте», к какому классу
они относятся, и какие техники защиты к ним применяются: **шифрование**,
**обфускация** и **обезличивание** (включая псевдонимизацию).

## Краткие определения

| Метод | Что делает | Когда применяется |
|-------|------------|-------------------|
| **Шифрование (encryption)** | Превращает данные в шифротекст с помощью ключа; обратимо при наличии ключа | Защита от утечек at-rest / in-transit; обязательное требование для УЗ-1 |
| **Обфускация (obfuscation / masking)** | Скрывает часть значения (XXXX-XXXX-1234) или заменяет на эквивалентные данные | UI, логи, отчёты, экраны кассиров |
| **Обезличивание (anonymization)** | Необратимое удаление связи с субъектом | Аналитика, BI, ML, исследовательские витрины |
| **Псевдонимизация (pseudonymization)** | Замена идентификатора на токен; обратимо через защищённую таблицу соответствия | Промежуточные слои обработки (CRM-аналитика) |
| **Токенизация** | Замена ценного значения (PAN, СНИЛС) на токен из защищённого хранилища | Платёжные данные |

## Уровни чувствительности (вводятся для тегирования, см. `tagging.md`)

| Уровень | Описание | Примеры |
|---------|----------|---------|
| `L1 — Public` | Публично доступная информация | Прейскурант, адрес клиники |
| `L2 — Internal` | Для внутреннего использования | Графики работы сотрудников, нормативы |
| `L3 — Confidential` | ПДн, финансовые, коммерческая тайна | Ф.И.О., телефон, оклады, цены поставщиков |
| `L4 — Restricted` | Спец. категории ПДн, врачебная тайна | Диагнозы, результаты анализов, мед. книжки |
| `L4+ — Highly Restricted` | Особо чувствительные | ВИЧ, психиатрия, наркология, генетика |

Легенда:
- **E (шифрование)** — `at-rest` (хранение) и/или `in-transit` (передача) и/или `field-level`.
- **M (обфускация / маскирование)** — частичное скрытие в UI, логах, отчётах.
- **A (обезличивание / псевдонимизация)** — для аналитики, ML и долгого хранения.

## Сводный реестр инструментов защиты

| Слой | Инструмент | Назначение |
|------|------------|------------|
| Управление ключами | **HashiCorp Vault** / **Yandex KMS** | Хранение мастер-ключей, ротация, аудит, envelope-encryption |
| Шифрование БД | **PostgreSQL + pgcrypto / TDE** | At-rest + field-level encryption |
| Шифрование объектов | **S3-совместимое хранилище + SSE-KMS** | Хранение сканов, PDF |
| Токенизация | **Vault Transform Secrets Engine** | PAN, СНИЛС, ИНН → токены |
| Шифрование канала | **TLS 1.3 / mTLS** | Все интеграции (ККМ↔1С, клиника↔лаборатория, клиент↔API) |
| Маскирование в UI | **React-маски / format-preserving masks** | Кассы, ресепшен, BI |
| Маскирование в логах | **Logback / serilog с фильтрами + Vector + Cribl** | Удаление/хеширование ПДн в логах |
| Обезличивание | **ARX Data Anonymization Tool** / **AWS Glue Anonymizer** | k-анонимизация, l-diversity, дифф. приватность для BI/ML |
| Аудит и контроль | **Elastic + Wazuh / Auditbeat** | SIEM, события доступа |
| DLP | **InfoWatch Traffic Monitor** / **Kaspersky DLP** | Контроль исходящего трафика, e-mail |
| Управление доступом | **Keycloak (RBAC + ABAC)** + **OPA / Cedar** | Роли, политики, контекстные правила |
| MFA | **Yandex.ID / TOTP / WebAuthn** | Все учётки с доступом к L3+ |
| Управление согласиями | **Custom Consent Service** + ЭП клиента | Сбор, отзыв согласий |
| Управление сроками | **Retention engine + cron-задачи** | Автоматическое удаление по политикам |
| Тегирование | **Apache Atlas / OpenMetadata** | Реестр данных и тегов чувствительности |

## Принципы выбора метода

```
                ┌─────────────────────────────────────────────────┐
                │ Нужны ли эти данные пользователю/системе целиком?│
                └─────────────────────────────────────────────────┘
                          │ да                       │ нет
                          ▼                          ▼
            ┌─────────────────────┐     ┌──────────────────────────┐
            │ Шифрование (KMS)    │     │ Можно ли восстановить    │
            │ (защита от утечек)  │     │ оригинал в будущем?      │
            └─────────────────────┘     └──────────────────────────┘
                                              │ да           │ нет
                                              ▼              ▼
                                  ┌─────────────────┐  ┌─────────────────┐
                                  │ Псевдонимизация │  │ Обезличивание   │
                                  │ / токенизация   │  │ (анонимизация)  │
                                  └─────────────────┘  └─────────────────┘

            Дополнительно: всегда применять маскирование в UI/логах
            для всех данных уровней L3 и выше.
```
