## Как читать диаграммы

Все DFD оформлены в нотации Mermaid (`graph LR`), которая поддерживает GitHub/GitLab «из
коробки». Условные обозначения:

- **Прямоугольник** — процесс (Process).
- **Прямоугольник с двойной линией** — внешний актор / источник или получатель (External Entity).
- **Цилиндр / параллелограмм** — хранилище данных (Data Store).
- Подпись на стрелке — категория передаваемых данных. К именам чувствительных
  данных добавлен префикс `[PII]` / `[МТ]` / `[ФИН]` (см. [`tagging.md`](tagging.md)).

Диаграммы в директории `diagrams/` отражают состояние As-Is. Доработанные диаграммы
с инструментами защиты на каждом этапе лежат в `diagrams_to_be/`.

## Краткий вывод по аудиту

Компания «Медикаменте» работает с большим объёмом персональных и медицинских данных,
но защищает их фактически только периметром локальной сети и доменной аутентификацией
Active Directory. Конфиденциальные данные хранятся в открытом виде в Excel/PDF/JPG
на общем сетевом диске, перемещаются между сотрудниками по электронной почте и в
1С без журналирования доступа. Это противоречит требованиям ФЗ-152 «О персональных
данных», 323-ФЗ «Об основах охраны здоровья граждан» (врачебная тайна) и приказу
ФСТЭК № 21. Конкретные проблемы и предложения по их устранению — в
[`problems.md`](problems.md), классификация и инструменты защиты — в
[`data_protection.md`](data_protection.md).
