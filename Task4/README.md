# Task 4 — Узкие места миграции монолита «Медикаменте» на микросервисы

В этой директории — анализ потенциальных узких мест миграции устаревшей
монолитной системы «Медикаменте» (Excel + 1С + общий диск + локальный
Windows-сервер) на микросервисную архитектуру, спроектированную в Task 2.

## Состав

| Файл | Назначение |
|------|------------|
| [`analysis.md`](analysis.md) | Анализ текущей системы и целей миграции, перечень затрагиваемых компонентов |
| [`ishikawa.drawio`](ishikawa.drawio) | **Диаграмма Исикавы** в формате draw.io / diagrams.net (XML) |
| [`ishikawa.md`](ishikawa.md) | Та же диаграмма в Mermaid + ASCII для просмотра в репозитории |
| [`recommendations.md`](recommendations.md) | Меры устранения узких мест с приоритетами и трудозатратами |
| [`report.md`](report.md) | Сводный отчёт: проблемы, причины, решения |

## Как открыть диаграмму

1. **draw.io** (рекомендуется):
   - Перейти на [app.diagrams.net](https://app.diagrams.net/);
   - File → Open from… → Device → выбрать `ishikawa.drawio`.
   - Альтернатива: открыть в VS Code через расширение «Draw.io Integration».
2. **Альтернатива в браузере** — открыть `ishikawa.md` прямо в GitHub/GitLab,
   там же есть Mermaid-визуализация и ASCII-представление.

## Подход к анализу

Узкие места рассматриваются по шести категориям (костям рыбы):

1. **Инфраструктура** — серверы, сеть, ЦОД.
2. **Программное обеспечение** — стеки, фреймворки, легаси-системы.
3. **Интеграция** — взаимодействие с 1С, ККМ, лабораторией, банком.
4. **Данные** — миграция, целостность, синхронизация, объёмы.
5. **Персонал** — компетенции, наём, обучение.
6. **Процессы** — управление, релизы, эксплуатация.

«Голова рыбы»: **Снижение производительности и стабильности системы при
миграции** — то, что мы хотим предотвратить.
