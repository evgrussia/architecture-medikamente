# Классификация данных «Медикаменте»

Документ описывает многомерную классификацию данных, которая используется как
вход для выбора методов и инструментов защиты в [`protection_tools.md`](protection_tools.md).

## Оси классификации

Каждый элемент данных (поле БД, объект в S3, событие Kafka, файл на диске) описывается
по пяти осям:

1. **Состояние** (state).
2. **Чувствительность** (sensitivity).
3. **Профиль риска** (risk profile).
4. **Объём и распространение** (scale).
5. **Правовой режим** (legal regime).

---

## 1. Состояние данных

| Код | Состояние | Что значит | Типовые риски |
|-----|-----------|-----------|----------------|
| **R**  | **At-Rest** | Данные в БД, на диске, в объектном хранилище, в бэкапе | Кража носителя, утечка дампа БД, незашифрованный бэкап |
| **T**  | **In-Transit** | Данные в сети — между сервисами, к клиенту, к контрагенту | Перехват, MITM, подмена |
| **U**  | **In-Use** | Данные в оперативной памяти, на экране, в логах | Memory dump, screenshot, утечка через логи, инсайдер |
| **A**  | **In-Archive** | Долгое хранение, бэкапы, экспорт для регулятора | Хищение носителя, отказ криптосистемы со временем |
| **D**  | **In-Deletion** | Процесс удаления | Неполное удаление, остаточные копии, отзыв согласия не доходит до всех мест |

Каждое состояние требует своего набора мер защиты — соответствие отражено в `protection_tools.md`.

---

## 2. Чувствительность (уровни)

Перенято из `Task1/data_protection.md`:

| Уровень | Описание | Примеры в «Медикаменте» |
|---------|----------|--------------------------|
| `L1 — Public` | Открытая информация | Прейскурант, адрес клиники, отзывы (после модерации) |
| `L2 — Internal` | Для внутреннего использования | Графики, нормативы, регламенты |
| `L3 — Confidential` | ПДн, финансовые, коммерческая тайна | ФИО, телефон, e-mail, оклады, цены поставщиков |
| `L4 — Restricted` | Спец. категории ПДн, врачебная тайна | Диагнозы (МКБ-10), анализы, заключения, мед. книжки |
| `L4+ — Highly Restricted` | Особо чувствительные | ВИЧ, психиатрия, наркология, генетика |

## 3. Профиль риска

Профиль риска = **вероятность инцидента × ущерб**. Используется матрица 3×3.

| Вероятность \ Ущерб | Низкий | Средний | Высокий |
|---------------------|--------|---------|---------|
| **Низкая**          | 🟢 RP1 | 🟢 RP2  | 🟡 RP3  |
| **Средняя**         | 🟢 RP2 | 🟡 RP3  | 🟠 RP4  |
| **Высокая**         | 🟡 RP3 | 🟠 RP4  | 🔴 RP5  |

Уровни:

- **RP1–RP2** — низкий. Минимальные базовые меры.
- **RP3** — средний. Базовые меры + аудит.
- **RP4** — высокий. Шифрование at-rest и in-transit, OPA, SIEM-алерты, DLP.
- **RP5** — критический. Дополнительные ключи KMS, изоляция домена, диф. приватность,
  break-the-glass + обязательный аудит и DPO-нотификация.

### Шкала ущерба для «Медикаменте»

| Уровень ущерба | Финансовый | Регуляторный | Репутационный |
|----------------|------------|--------------|----------------|
| Низкий | < 1 млн ₽ | Предписание | Местные новости |
| Средний | 1–10 млн ₽ | Штраф РКН до 700 тыс. ₽ | Региональные СМИ |
| Высокий | > 10 млн ₽ | Оборотный штраф 152-ФЗ (1–3% выручки) или приостановка деятельности | Федеральные СМИ, отзыв лицензии |

---

## 4. Объём и распространение

| Код | Объём | Применение |
|-----|-------|------------|
| **S** (Single) | Один субъект | Личный кабинет |
| **M** (Many) | До 10⁴ субъектов | Один филиал |
| **L** (Large) | 10⁴–10⁶ субъектов | Сеть клиник после роста |
| **XL** (Massive) | > 10⁶ субъектов | Перспективные интеграции (государственные сервисы) |

При L и XL автоматически применяется требование УЗ-1 по ПП-1119 (более 100 000
субъектов спец. категорий → УЗ-1).

---

## 5. Правовой режим

| Код | Регулирование | Особенности |
|-----|---------------|-------------|
| `PII-152` | 152-ФЗ «О персональных данных» | Согласия, локализация в РФ, право на удаление |
| `MED-323` | 323-ФЗ «Об основах охраны здоровья» | Врачебная тайна, ограниченный круг лиц |
| `FIN-86` | 86-ФЗ «О ЦБ РФ» + ГОСТ Р 57580 | Защита платёжных операций |
| `COMM-98` | 98-ФЗ «О коммерческой тайне» | Режим ком. тайны, договоры, NDA |
| `LAB-323-DPA` | 152-ФЗ + договор обработки | Передача в лабораторию по DPA |
| `HR-TK` | Трудовой кодекс + 152-ФЗ | Кадровые данные работников |
| `OFD-FNS` | 54-ФЗ + НК РФ | Фискальные данные |

## Выводы

1. **Львиная доля данных** в «Медикаменте» — это L3/L4 с риском RP4-RP5, что
   автоматически требует УЗ-1 и применения всего арсенала средств защиты.
2. **L4+ класс** (ВИЧ, психиатрия) выделен в отдельный домен с собственным
   жизненным циклом ключей и ABAC.
3. **Аудит-журналы** сами по себе несут чувствительные идентификаторы — их
   защита и ретеншн (3 года минимум по 187-ФЗ КИИ для значимых субъектов)
   проектируется так же серьёзно, как и медкарт.
4. **Аналитические данные после анонимизации** опускаются в L2-L3 — это
   возможно только при условии корректной работы Anonymization Pipeline.
