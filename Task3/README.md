# Task 3 — Стратегия защиты данных «Медикаменте»

В этой директории — стратегия защиты данных, охватывающая все состояния
(at-rest, in-transit, in-use), классификацию по чувствительности и профилю
риска, инструменты защиты и средства автоматизированного контроля.

## Принципы стратегии

1. **Защита data-in-three-states**:
   - **At-rest** — TDE, SSE-KMS, field-level encryption через KMS-envelope,
     crypto-shredding при удалении.
   - **In-transit** — TLS 1.3 на внешних каналах, mTLS на внутренних, JWS
     для подписи документов.
   - **In-use** — маскирование в UI, маскирование в логах, минимальный TTL
     data-key в RAM, защищённые анклавы для самых чувствительных операций.
2. **Tag-driven защита**. Тег → политика → инструмент. Без тегов поле
   считается `sensitivity:l3` по умолчанию.
3. **Defense-in-depth**: периметр (FW, WAF), идентификация (ЕСИА, MFA),
   авторизация (OPA), шифрование, аудит (SIEM), DLP, реактивные меры
   (Retention, инциденты).
4. **Risk-based**: чем выше профиль риска данных, тем дороже и жёстче
   инструменты.
5. **Compliance**: 152-ФЗ (УЗ-1), 323-ФЗ (врачебная тайна), ФСТЭК № 21,
   ПП-1119, ГОСТ Р 57580 (касса), 98-ФЗ (коммерческая тайна), требования
   ЦБ/ОФД.
