# Реестр программных и аппаратных средств защиты данных

Документ — единый реестр инструментов, необходимых для реализации стратегии
из [`protection_tools.md`](protection_tools.md) и автоматизированного контроля
из [`automated_controls.md`](automated_controls.md). Используется как чек-лист
для закупки/внедрения и как реестр СЗИ для отчётов в ФСТЭК/РКН.

## 5. Сводный CAPEX-/OPEX-блок (для понимания порядка)

Точная стоимость зависит от объёмов лицензий, числа филиалов и подрядчиков.
Для MVP-периода планируется следующий **порядок** затрат:

| Категория | Тип | Порядок (млн ₽/год) |
|-----------|-----|---------------------|
| Сертифицированные СЗИ ФСТЭК (UserGate, Kaspersky, InfoWatch) | OPEX | 3–5 |
| Vault Enterprise + HSM | CAPEX + поддержка | 4–6 |
| КриптоПро/Континент/токены ЭЦП | CAPEX | 1–2 |
| Подписка Yandex Cloud (KMS, Object Storage, DataLens) | OPEX | 2–4 |
| Серверная инфраструктура | CAPEX | 8–12 |
| Команда внедрения и поддержки (SecOps + DPO) | OPEX | 6–10 |

## Итог

Реестр охватывает **всю стратегию защиты**: ключи, шифрование, доступ, аудит,
DLP, каталог, аналитика с PbD, CI/CD-гейты и аппаратные средства от HSM до
шредеров. Каждый продукт привязан к конкретной задаче из
[`protection_tools.md`](protection_tools.md) и/или
[`automated_controls.md`](automated_controls.md). Реестр готов к использованию
как:

- чек-лист для закупки и внедрения;
- реестр СЗИ для отчётов в ФСТЭК/РКН;
- источник данных для PIA и инвентаризации СЗИ.
