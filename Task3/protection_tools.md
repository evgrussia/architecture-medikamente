# Главная таблица — данные, обоснование защиты, инструменты

Эта таблица отвечает на вопрос задания «**Данные → В чём необходимость защиты →
Инструменты защиты**». Применяется к классам данных из
[`data_classification.md`](data_classification.md).

## Условные обозначения столбца «Инструменты защиты»

| Сокр. | Расшифровка | Назначение |
|-------|-------------|------------|
| TDE | Transparent Data Encryption (PostgreSQL) | Шифрование at-rest на уровне таблиц |
| FLE | Field-Level Encryption (pgcrypto + KMS envelope) | Шифрование отдельных полей |
| SSE-KMS | S3 Server-Side Encryption (per-bucket / per-object ключи KMS) | Шифрование объектов в S3 |
| RLS | Row-Level Security в PostgreSQL | Изоляция строк по пациенту/арендатору |
| KMS | HashiCorp Vault / Yandex KMS | Управление ключами + ротация |
| TOK | Vault Transform / токенизация | Замена ценного значения на токен |
| TLS | TLS 1.3 на внешнем периметре | Шифрование канала |
| mTLS | Mutual TLS внутри / с партнёрами | Двусторонняя аутентификация канала |
| JWS | JSON Web Signature | Подпись документов и сообщений |
| MASK-UI | Маскирование в UI | Скрытие части значения на экране |
| MASK-LOG | Маскирование в логах | Redact / hash в логах |
| PSEUDO | Псевдонимизация | Обратимая замена на токен для аналитики |
| ANON | Анонимизация (k-anon, l-div, diff-priv) | Необратимая обработка для BI/ML |
| OPA | Open Policy Agent (PDP) | ABAC по тегам |
| RBAC | Роли | Базовая модель доступа |
| ABAC | Атрибутный доступ | На основе контекста |
| MFA | Многофакторная аутентификация | Защита учётных записей |
| WAF | Web Application Firewall | Защита приложений |
| DLP | Data Loss Prevention | Контроль исходящих данных |
| SIEM | Elastic + Wazuh | Журналирование, корреляция, алерты |
| RET | Retention Engine | Удаление по таймеру / запросу |
| CONS | Consent Service | Согласия и право на удаление |
| CAT | Apache Atlas / OpenMetadata | Каталог данных + теги |
| HSM | Hardware Security Module | Аппаратное хранение ключей |
| MDM | Mobile Device Management | Контроль рабочих устройств |
| NAC | Network Access Control | Допуск в сеть |

## Краткий итог по слоям защиты

### Что защищает данные **at-rest**

| Слой | Инструмент | Где |
|------|------------|-----|
| Диск/том | LUKS / BitLocker | Серверы, рабочие станции |
| СУБД | PostgreSQL TDE | Все БД |
| Поле | pgcrypto + KMS envelope | L4 / L4+ поля |
| Объект | S3 SSE-KMS | Сканы, PDF |
| Бэкап | Отдельный backup-ключ KMS | Off-site |
| Ключи | HSM / KMS | KMS / Vault |

### Что защищает данные **in-transit**

| Уровень | Инструмент | Где |
|---------|------------|-----|
| Внешний периметр | TLS 1.3 + WAF | API GW, Web, моб. |
| Сервис ↔ сервис | mTLS через service mesh (istio) | Внутри кластера |
| ККМ ↔ 1С | TLS-туннель + сертификат | Касса |
| Сервис ↔ внешний контрагент | mTLS + JWS | Лаборатория, банк, поставщики |
| Уведомления | TLS к SMS/Push/Email GW + маска | Notification Service |
| Шина событий | mTLS + ACL | Kafka |

### Что защищает данные **in-use**

| Уровень | Инструмент | Где |
|---------|------------|-----|
| UI | Маскирование, watermark | Веб / моб. |
| Лог | Redact / hash по тегам | Все сервисы |
| Память | Короткий TTL data-key, secure-string | KMS-клиенты |
| Доступ | OPA ABAC + JIT-grant + break-the-glass | API GW |
| Устройство | MDM, шифрование диска ноутбука | Все рабочие станции |
| Печать | Pull-printing + аутентификация картой | Принтеры |

### Что защищает данные **in-deletion** и **in-archive**

| Уровень | Инструмент | Где |
|---------|------------|-----|
| Удаление | Retention Engine + crypto-shredding | Все БД, S3, бэкапы, шина |
| Архив | Off-site копия в РФ-ЦОДе, отдельный ключ | Бэкап |
| Подтверждение удаления | DPO-Console + аудит | DPO workflow |

## Кратко о подходе

Каждое поле в системе **тегируется** в Data Catalog (см. `Task1/tagging.md`).
Платформа защиты читает теги и **автоматически выбирает** нужный комбо-инструмент
из таблицы выше — разработчику бизнес-сервиса не нужно явно реализовывать
шифрование, токенизацию, маски, политики ретеншена.

Это и есть **«реализация базовых инструментов защиты»** из формулировки задания:
инструменты вшиты в платформу, бизнес-код использует их декларативно через теги.
